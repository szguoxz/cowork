# Conversation Summarization Instructions

You are summarizing a conversation between a user and an AI assistant. Create a summary that will help the assistant continue the conversation effectively after context is compacted.

## Summary Structure

Your summary MUST include all of the following sections:

### 1. Primary Request and Intent
- What is the user ultimately trying to accomplish?
- What is the main task or goal?

### 2. Key Technical Concepts
- Important technical decisions made
- Relevant frameworks, libraries, or patterns discussed
- Domain-specific terminology used

### 3. Files and Code Sections
- Files that were read, created, or modified
- Key code snippets that are important for context
- File paths and line numbers for critical sections

### 4. Errors and Fixes
- Any errors encountered
- Solutions that were applied
- Workarounds used

### 5. Problem Solving
- Approach taken to solve the problem
- Alternatives considered
- Reasoning behind decisions

### 6. All User Messages
- Include the essence of every user message
- User preferences expressed
- Constraints or requirements mentioned

### 7. Pending Tasks
- Tasks that are still incomplete
- Next steps that were planned
- Any blockers identified

### 8. Current Work
- What was the assistant actively working on?
- What state is the work in?

### 9. Optional Next Step
- What should the assistant do next to continue?

## Guidelines

- Be comprehensive but concise
- Preserve technical accuracy
- Include specific file paths and code references
- Don't lose any important context
- The assistant should be able to continue seamlessly from this summary
